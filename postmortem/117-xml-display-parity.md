# 117 - XML Display Parity

## Background

XML values in result cells used to collapse to `<XML>` when they were longer than the cell width.  JSON values already used a more useful model: short JSON renders inline, while long JSON shows a raw prefix with an ellipsis.

The XML value viewer also runs `xmllint --format -` when available.  `xmllint` adds `<?xml version="1.0"?>` when formatting XML that did not include a declaration, so the viewer could show text that was not part of the stored value.

## Decision

XML result cells now follow the same truncation model as JSON: short values can render inline with lightweight token faces, while long values show the raw prefix and use `...`/ellipsis when the value does not fit.  The grid does not run `nxml-mode` or buffer font-lock for cell rendering; it only applies a small scanner to fully visible XML values, with a fixed length cap.

The XML viewer strips an XML declaration only when it was generated by `xmllint`.  If the stored value already starts with an XML declaration, the viewer preserves it.
